# Molink 本地开发启动指南

本文档旨在指导开发者如何在本地环境中（非 Docker 容器内）启动 Molink 的前后端服务，以便进行开发和调试。

## 前置条件

在开始之前，请确保您的开发环境已安装以下软件：

*   **Python 3.11+** (推荐 3.13)
*   **Node.js 18+** & **npm**
*   **Docker Desktop** (仅用于运行 MySQL 和 Redis 服务)

## 1. 启动基础服务 (MySQL & Redis)

我们使用 Docker Compose 来快速启动数据库和缓存服务。

1.  打开终端，进入项目根目录的 `molink-local` 文件夹：
    ```bash
    cd molink-backend
    ```

2.  启动 MySQL 和 Redis 服务：
    ```bash
    docker-compose up -d mysql redis
    ```
    > **注意**：我们只启动 `mysql` 和 `redis` 服务，**不启动** `backend` 和 `frontend` 服务，因为我们将在本地运行它们。

3.  验证服务状态：
    确保 MySQL (端口 3306) 和 Redis (端口 16379) 正在运行。
    *   MySQL: `localhost:3306`
    *   Redis: `localhost:16379` (注意：本地映射端口为 16379，以防与本地已有的 Redis 冲突)

## 2. 启动后端 (Backend)

### 2.1 环境配置

由于本地开发需要连接 `localhost` 而不是 Docker 容器名，我们需要确保配置文件正确。

1.  **检查配置文件** (`molink-local/backend/app/core/config.py`)：
    确保配置类中的数据库和 Redis 主机默认为 `localhost`，并且 Redis 端口为 `16379`。

    ```python
    # molink-local/backend/app/core/config.py

    # ...
    # 数据库配置
    MYSQL_HOST: str = "localhost"  # 确保这里是 localhost
    MYSQL_PORT: int = 3306
    # ...
    
    # Redis配置
    REDIS_HOST: str = "localhost"  # 确保这里是 localhost
    REDIS_PORT: int = 16379        # 确保这里是 16379 (对应 Docker 映射端口)
    # ...
    ```
    > 提示：现在的代码库已经包含了自动加载 `.env` 的逻辑，所以通常不需要手动修改代码，除非您想更改默认值。

2.  **安装依赖**：
    进入后端目录并安装依赖（建议使用虚拟环境）：
    ```bash
    cd molink-local/backend
    # 如果使用 venv
    python -m venv venv
    # Windows 激活
    .\venv\Scripts\activate
    # Mac/Linux 激活
    # source venv/bin/activate
    
    pip install -r requirements.txt
    ```

### 2.2 启动服务

在 `molink-local/backend` 目录下运行：

```bash
python app/main.py
```

或者使用 PyCharm 等 IDE 直接运行 `app/main.py`。

成功启动后，您将看到类似以下的日志：
```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

*   **API 文档地址**: [http://localhost:8000/api/docs](http://localhost:8000/api/docs)

## 3. 启动前端 (Frontend)

### 3.1 环境配置

进入前端目录：
```bash
cd molink-local/frontend
```

### 3.2 安装依赖

```bash
npm install
```

### 3.3 启动开发服务器

```bash
npm run dev
```

启动成功后，终端会显示访问地址，通常为：
*   **前端访问地址**: [http://localhost:5173](http://localhost:5173)

## 4. 常见问题排查

### 4.1 数据库连接失败 (OperationalError: 2003)
*   **原因**: 后端尝试连接的主机名不正确，或者 Docker 中的 MySQL 未启动。
*   **解决**:
    1.  确保 Docker 容器 `molink-mysql` 正在运行。
    2.  检查 `config.py` 或环境变量中 `MYSQL_HOST` 是否为 `localhost`。

### 4.2 ModuleNotFoundError: No module named 'app.core'
*   **原因**: Python 运行路径不正确。
*   **解决**: 确保在 `backend` 目录下运行 `python app/main.py`。代码中已添加路径修复逻辑，如果仍报错，请检查 IDE 的 Working Directory 是否设置为 `molink-local/backend`。

### 4.3 端口冲突
*   **MySQL (3306)**: 如果本地已经安装并运行了 MySQL，可能会冲突。请停止本地 MySQL 或修改 docker-compose 中的端口映射。
*   **Redis (16379)**: 使用了非标准端口以避免冲突。确保后端配置的 `REDIS_PORT` 与 docker-compose 映射的端口一致。

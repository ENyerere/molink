# Molink-App 启动指南

## 🚀 快速启动 (推荐方法)

### 使用智能启动脚本（自动修复错误）

```powershell
cd D:\WorkSpace\package\molink-local
.\smart-start.ps1
```

### 或使用标准启动脚本

```powershell
cd D:\WorkSpace\package\molink-local
.\start.ps1
```

## 📋 详细启动步骤

### 1. 环境要求

确保您的系统已安装以下软件：

- **Docker Desktop** 20.10+ 
- **Docker Compose** 2.0+
- **PowerShell** 5.1+ (Windows)
- **Git** (可选，用于下载项目)

```powershell
# 检查版本
docker --version
docker-compose --version
```

### 2. 项目下载和解压

如果您有项目压缩包：

```bash
# 解压项目
tar -xzf molink-local.tar.gz
cd molink-local

# 或者如果从Git仓库获取
git clone <repository-url>
cd molink-local
```

### 3. 检查项目结构

确保以下文件和目录存在：

```
molink-local/
├── docker-compose.yml        # Docker编排配置 ✅
├── README.md                 # 项目文档 ✅
├── 启动指南.md               # 本文档 ✅
├── init-db.sql              # 数据库初始化脚本 ✅
├── scripts/                  # 运维脚本目录 ✅
├── backend/                  # 后端项目 ✅
├── frontend/                 # 前端项目 ✅
└── uploads/                  # 文件上传目录 ✅
```

### 4. 配置环境变量 (生产环境推荐)

编辑 `docker-compose.yml` 文件，修改敏感配置：

```yaml
# 修改数据库密码
MYSQL_ROOT_PASSWORD: your_secure_root_password
MYSQL_PASSWORD: your_secure_user_password

# 修改应用密钥
SECRET_KEY: your_super_secret_key_at_least_32_characters
```

### 5. 启动服务

#### Windows 环境启动 (推荐)

**方法一：PowerShell (推荐)**

```powershell
# 进入项目根目录
cd D:\WorkSpace\package\molink-local

# 运行环境检查 (可选)
.\check-env.ps1

# 启动服务
.\start.ps1
```

**方法二：CMD 命令提示符**

```cmd
# 进入项目根目录
cd D:\WorkSpace\package\molink-local

# 运行环境检查 (可选)
check-env.bat

# 启动服务
start.bat
```

**方法三：双击文件**

- 双击 `start.ps1` (PowerShell)
- 双击 `start.bat` (CMD)

**⚠️ PowerShell 用户注意:**
如果遇到 "找不到命令" 错误，请使用 `.\` 前缀：
```powershell
.\check-env.ps1
.\start.ps1
```

#### Linux/macOS 环境启动

**方法一：使用启动脚本 (推荐)**

```bash
# 进入项目目录
cd molink-local

# 赋予执行权限
chmod +x scripts/*.sh

# 执行启动脚本
./scripts/start.sh
```

#### 方法二：手动启动

```bash
# 构建并启动所有服务
docker-compose up -d --build

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 6. 验证服务启动

启动完成后，等待 30-60 秒让所有服务完全启动。

#### 检查服务状态

```bash
# 查看所有容器状态
docker-compose ps

# 应该看到以下容器都处于 "Up" 状态：
# - molink-mysql    (MySQL数据库)
# - molink-redis    (Redis缓存)
# - molink-backend  (FastAPI后端)
# - molink-frontend (Nginx前端)
```

#### 健康检查

```bash
# 检查后端API
curl http://localhost:8000/health

# 检查前端
curl -I http://localhost

# 检查数据库连接
docker exec molink-mysql mysql -uroot -proot_password -e "SELECT 'Database OK' as status;"
```

### 7. 首次访问

#### 访问地址

- **前端界面**: http://localhost
- **API文档**: http://localhost:8000/api/docs
- **API健康检查**: http://localhost:8000/health

#### 管理员账户登录

- **邮箱**: `admin@molink.local`
- **密码**: `admin123`

**⚠️ 重要提醒**: 首次登录后请立即修改默认密码！

### 8. 创建用户账户

1. 打开浏览器访问: http://localhost
2. 点击"注册"按钮
3. 填写用户信息并创建账户
4. 使用新账户登录开始使用

## 🔧 常用运维命令

### 启动/停止服务

```bash
# 启动服务
./scripts/start.sh
# 或
docker-compose up -d

# 停止服务
./scripts/stop.sh
# 或
docker-compose down

# 重启服务
docker-compose restart
```

### 查看日志

```bash
# 查看所有服务日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f frontend
docker-compose logs -f mysql
```

### 数据管理

```bash
# 备份数据
./scripts/backup.sh

# 查看数据库
docker exec -it molink-mysql mysql -uroot -proot_password molink_db

# 进入后端容器
docker exec -it molink-backend bash

# 进入前端容器
docker exec -it molink-frontend sh
```

### 清理和重建

```bash
# 停止并删除所有容器和数据
docker-compose down -v

# 清理Docker缓存 (可选)
docker system prune -f

# 完全重建
docker-compose up -d --build --force-recreate
```

## 🚨 故障排除

### 🔧 TypeScript编译错误 (常见)

**症状**: 前端构建失败，错误信息包含：
```
error TS6133: 'User' is declared but its value is never read.
error TS2503: Cannot find namespace 'NodeJS'.
```

**解决方案**:
```powershell
# 方案1：使用智能启动脚本（推荐）
.\smart-start.ps1

# 方案2：手动修复
cd frontend
# 删除未使用的导入和变量（已在smart-start.ps1中自动处理）
```

### 🌐 网络连接问题 (npm安装失败)

**症状**: npm网络错误
```
npm error code ECONNRESET
npm error network request failed
```

**解决方案**:
```powershell
# 切换npm镜像源
.\set-npm-source.ps1 -Source npmmirror

# 或选择其他镜像源
.\set-npm-source.ps1 -Source cnpmjs
.\set-npm-source.ps1 -Source taobao
.\set-npm-source.ps1 -Source official

# 然后重新构建
docker-compose build frontend --no-cache
```

### 🔧 容器启动失败

**症状**: 某些容器显示 "Exited" 状态

**解决方案**:
```bash
# 查看详细错误日志
docker-compose logs [service-name]

# 检查端口占用
netstat -tulpn | grep :80
netstat -tulpn | grep :8000
netstat -tulpn | grep :3306

# 重启失败的容器
docker-compose restart [service-name]
```

### 数据库连接失败

**症状**: 后端服务无法连接数据库

**解决方案**:
```bash
# 检查MySQL容器状态
docker-compose ps mysql

# 手动启动MySQL
docker-compose up mysql -d

# 检查MySQL日志
docker-compose logs mysql

# 等待MySQL完全启动 (约30秒)
sleep 30
```

### 前端页面无法访问

**症状**: 浏览器无法打开 http://localhost

**解决方案**:
```bash
# 检查前端容器状态
docker-compose ps frontend

# 重新构建前端
docker-compose up frontend -d --build

# 检查端口映射
netstat -tulpn | grep :80
```

### npm 网络连接错误 (ECONNRESET)

**症状**: 
```
npm error code ECONNRESET
network request to https://registry.npmjs.org/ failed
Client network socket disconnected before secure TLS connection was established
```

**解决方案:**

**方法一: 使用离线构建 (Windows推荐)**
```cmd
cd D:\WorkSpace\package\molink-local
offline-build.bat
```

**方法二: 手动配置镜像源**
```cmd
cd D:\WorkSpace\package\molink-local\frontend
npm config set registry https://registry.npmmirror.com/
npm install --registry=https://registry.npmmirror.com/
npm run build
```

**方法三: PowerShell 脚本**
已修复的 Dockerfile 自动使用国内镜像源，重新运行:
```powershell
.\start.ps1
```

**方法四: Docker 清理重建**
```cmd
# 清理 Docker 缓存
docker system prune -f

# 重新构建
docker-compose up -d --build
```
netstat -tulpn | grep :80
```

### API调用失败

**症状**: 前端登录/注册失败

**解决方案**:
```bash
# 检查后端服务状态
docker-compose ps backend

# 检查后端日志
docker-compose logs backend

# 测试API端点
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@molink.local","password":"admin123"}'
```

### 权限错误

**症状**: 文件权限问题

**解决方案**:
```bash
# 修复脚本权限
chmod +x scripts/*.sh

# 修复上传目录权限
sudo chown -R $USER:$USER uploads/
chmod -R 755 uploads/
```

### 内存不足

**症状**: 容器启动后立即退出

**解决方案**:
```bash
# 检查系统资源
free -h
df -h

# 清理Docker资源
docker system prune -f

# 减少MySQL内存使用
# 编辑 docker-compose.yml，在mysql服务的command中添加:
# --innodb-buffer-pool-size=128M
```

## 🚨 Windows 环境故障排除

### 常见问题及解决方案

#### 问题1: "找不到命令" 或 "command not found"

**PowerShell 环境:**
```
找不到命令: The term 'start.bat' is not recognized as a name...
```

**解决方案:**
- 使用 `.\` 前缀运行脚本：
  ```powershell
  .\check-env.ps1
  .\start.ps1
  ```
- 或切换到 CMD 命令提示符运行：
  ```cmd
  check-env.bat
  start.bat
  ```

**CMD 环境:**
```
找不到命令: 'sh' 不是内部或外部命令
```

**解决方案:**
- 不要运行 `.sh` 文件
- 使用 Windows 批处理文件：
  ```cmd
  check-env.bat
  start.bat
  ```

#### 问题2: "no configuration file provided: not found"

**可能原因:**
1. 没有在项目根目录运行脚本
2. `docker-compose.yml` 文件缺失或损坏
3. Docker 环境配置问题

**解决方案:**
```cmd
# 确保在项目根目录
cd D:\WorkSpace\package\molink-local

# 检查文件是否存在
dir docker-compose.yml

# 使用 Windows 启动脚本
start.bat
```

#### 问题3: Docker 未启动或无权限

**解决方案:**
1. 启动 Docker Desktop
2. 右键以管理员身份运行命令提示符
3. 检查 Docker 服务状态: `docker info`

#### 问题4: 端口被占用

**检查命令:**
```cmd
# 检查端口占用
netstat -an | find ":80"
netstat -an | find ":8000"
netstat -an | find ":3306"
netstat -an | find ":6379"
```

**解决方案:**
- 关闭占用端口的进程
- 或修改 docker-compose.yml 中的端口映射

#### 问题5: 权限拒绝错误

**解决方案:**
```cmd
# 以管理员身份运行命令提示符
# 或
# 右键项目文件夹 → 属性 → 安全 → 完全控制权限
```

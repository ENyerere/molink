# Molink 开发计划

> 本文档用于规划 Molink 项目未来需要实现的功能，按优先级和模块分类排列。当前基于项目现有代码结构（FastAPI + React + MySQL）提出建议。
>
> 状态说明：🔴 高优先级 / 🟡 中优先级 / 🟢 低优先级

---

## 一、当前已实现功能（Baseline）

在规划新功能前，先确认已有能力：

- [x] 用户注册 / 登录 / JWT 认证
- [x] 管理员后台（统计、用户管理、系统健康）
- [x] 工作空间 CRUD
- [x] 页面树形管理（父子嵌套、收藏、归档）
- [x] 块级编辑器（text / h1-h6 / ul / ol / image / code / quote / table）
- [x] 数据库功能（字段定义、记录存储、多视图枚举）
- [x] 文件上传 / 管理
- [x] WebSocket 实时协作（光标同步、内容广播）
- [x] Docker 部署支持

---

## 二、核心体验（高优先级 🔴）

这些功能是用户日常使用的刚需，缺失会严重影响产品可用性。

### 2.1 权限与协作模型

**现状问题**：目前只有 `owner_id` 一种权限判断，没有成员、访客、只读等角色。

建议实现：

| 功能 | 说明 |
|------|------|
| **工作空间成员管理** | 邀请其他用户加入工作空间，分配角色 |
| **角色体系** | Owner（所有者）/ Editor（编辑者）/ Viewer（只读者） |
| **页面级权限覆盖** | 允许单个页面设置与空间不同的权限 |
| **公开分享链接** | 生成只读或可编辑的公开链接，支持密码保护 |

**数据层建议**：新增 `workspace_members` 关联表（user_id + workspace_id + role）。

---

### 2.2 全文搜索

**现状问题**：没有搜索功能，页面和块多了之后完全找不到内容。

建议实现：

| 功能 | 说明 |
|------|------|
| **页面标题搜索** | 最简单的起步，LIKE 查询即可 |
| **内容块全文搜索** | 搜索 blocks.content 中的文本 |
| **快捷键唤醒** | `Ctrl + K` 或 `Cmd + K` 快速搜索 |
| **搜索结果高亮** | 显示匹配片段和所在页面路径 |

**技术建议**：
- 初期可用 MySQL `FULLTEXT` 索引实现
- 后期可迁移到 Elasticsearch 或 Meilisearch 提升体验

---

### 2.3 页面历史版本

**现状问题**：用户误删或误改内容后无法恢复。

建议实现：

| 功能 | 说明 |
|------|------|
| **自动保存历史** | 每次页面更新时，将当前 blocks 快照存入 `page_versions` 表 |
| **版本时间线** | 页面右侧显示历史版本列表（类似 Notion 的 Page History） |
| **版本对比** | 对比两个版本之间的内容差异 |
| **一键回滚** | 恢复到任意历史版本 |

**数据层建议**：新增 `page_versions` 表（page_id + blocks_snapshot JSON + created_by + created_at）。

---

### 2.4 回收站（软删除）

**现状问题**：删除页面/块是物理删除，误操作无法挽回。

建议实现：

| 功能 | 说明 |
|------|------|
| **软删除机制** | 所有表添加 `deleted_at` 字段，删除时只标记时间戳 |
| **回收站页面** | 侧边栏增加"回收站"入口，展示近期删除的页面 |
| **恢复功能** | 从回收站恢复页面到原位置 |
| **自动清理** | 30 天后自动物理删除，释放空间 |

---

### 2.5 块级评论

**现状问题**：协作时无法对具体内容进行讨论。

建议实现：

| 功能 | 说明 |
|------|------|
| **选中文本评论** | 选中块内文字，弹出"添加评论" |
| **评论线程** | 每条评论支持回复，形成讨论串 |
| **评论状态** | 标记为"已解决"后折叠 |
| **@提及** | `@用户名` 通知被提及用户 |

**数据层建议**：新增 `comments` 表（block_id + user_id + content + parent_id + resolved_at）。

---

## 三、协作增强（中优先级 🟡）

这些功能能显著提升团队协作体验，但用户在没有核心功能时不会优先使用。

### 3.1 通知系统

| 功能 | 触发场景 |
|------|---------|
| **被邀请加入空间** | 收到工作空间邀请 |
| **被@提及** | 评论或页面内容中 `@用户名` |
| **页面被编辑** | 自己创建的页面被他人修改 |
| **评论回复** | 自己的评论收到回复 |

**实现建议**：
- 数据层：`notifications` 表（user_id + type + content + link + is_read + created_at）
- 展示层：前端右上角铃铛图标，红点提示未读数
- 推送层：后期可接入 WebSocket 实时推送

---

### 3.2 模板系统

| 功能 | 说明 |
|------|------|
| **内置模板** | 提供常见模板（项目管理、周记、会议纪要、任务看板） |
| **用户自定义模板** | 将当前页面保存为模板，供自己或团队复用 |
| **数据库模板** | 创建数据库时选择预设字段组合 |

---

### 3.3 导入 / 导出

| 功能 | 说明 |
|------|------|
| **导出 Markdown** | 单页面导出为 `.md` 文件 |
| **导出 HTML / PDF** | 用于分享或打印 |
| **导入 Markdown** | 将 `.md` 文件转为页面内容块 |
| **Notion 导入** | 长期目标，支持导入 Notion 导出的 ZIP/Markdown |

---

### 3.4 多人协作增强

**现状问题**：WebSocket 已实现基础广播，但缺少冲突处理。

| 功能 | 说明 |
|------|------|
| **操作锁** | 同一 block 正在被编辑时，其他用户只读 |
| **冲突提示** | 两人同时编辑同一 block 时，提示冲突并手动合并 |
| **用户头像浮层** | 页面上实时显示其他协作者的头像和光标位置 |
| **操作历史** | 显示"某某用户在 2 分钟前编辑了此块" |

---

### 3.5 移动端适配

**现状问题**：前端是桌面端布局，手机访问体验差。

| 功能 | 说明 |
|------|------|
| **响应式布局** | 侧边栏折叠、编辑器适配窄屏 |
| **移动端编辑器** | 简化块操作，支持触摸手势 |
| **PWA 支持** | 可添加到手机桌面，离线查看已加载内容 |

## 四、高级功能（低优先级 🟢）

这些功能锦上添花，建议等核心功能稳定后再投入开发。

### 4.1 AI 辅助

| 功能 | 说明 |
|------|------|
| **AI 续写** | 按 `Space` 或快捷键触发 AI 续写当前段落 |
| **内容总结** | 选中大段文字，一键生成摘要 |
| **语法优化** | 润色、纠错、翻译 |
| **数据库智能填充** | 根据字段内容自动补全或分类 |

---

### 4.2 API 与集成

| 功能 | 说明 |
|------|------|
| **API Token 管理** | 用户可生成 Token，通过 REST API 读写自己的数据 |
| **Webhook** | 页面更新、数据库变更时向外部 URL 推送事件 |
| **第三方登录** | 微信、GitHub、Google OAuth |
| **Slack / 钉钉机器人** | 通知推送到 IM 工具 |

---

### 4.3 数据可视化

| 功能 | 说明 |
|------|------|
| **个人仪表盘** | 最近编辑、待办统计、活跃趋势 |
| **团队仪表盘** | 成员贡献度、页面访问量、协作活跃度 |
| **数据库图表** | 将数据库记录转为饼图、柱状图、折线图 |

---

### 4.4 插件/扩展系统

| 功能 | 说明 |
|------|------|
| **自定义块类型** | 开发者可注册新的 block_type（如嵌入 Bilibili 视频、高德地图） |
| **主题市场** | 切换编辑器配色、字体、页面宽 |
| **快捷键自定义** | 用户可修改或新增快捷键 |

## 五、技术优化（持续推进）

这些不是用户可见的功能，但直接影响开发效率和系统稳定性。

### 5.1 数据库

| 优化项 | 说明 |
|--------|------|
| **Alembic 迁移** | 用 Alembic 管理数据库版本，替代现在的 `create_all()` |
| **软删除统一处理** | 所有查询默认过滤 `deleted_at IS NULL` |
| **连接池监控** | 暴露 SQLAlchemy 连接池指标到 `/health` |

### 5.2 后端架构

| 优化项 | 说明 |
|--------|------|
| **services 层补全** | 把路由层里的业务逻辑抽到 `services/` 和 `crud/` |
| **全局异常处理** | 统一返回 `{code, message, detail}` 格式的错误响应 |
| **请求日志** | 记录 API 调用耗时、用户、参数，便于排查问题 |
| **Rate Limiting** | 登录、注册接口加限流，防止暴力破解 |
| **单元测试** | 用 pytest + httpx 覆盖核心 API |

### 5.3 前端

| 优化项 | 说明 |
|--------|------|
| **虚拟滚动** | 长页面的块列表用虚拟滚动，避免 DOM 过多卡顿 |
| **懒加载图片** | 图片块进入视口才加载 |
| **离线编辑缓存** | 网络断开时把操作缓存到 LocalStorage，恢复后批量同步 |
| **打包优化** | Vite 按需加载、代码分割，减少首屏体积 |

### 5.4 部署与运维

| 优化项 | 说明 |
|--------|------|
| **CI/CD** | GitHub Actions 自动测试、构建 Docker 镜像 |
| **日志收集** | 后端日志输出到文件或 ELK/Loki |
| **监控告警** | 接入 Prometheus + Grafana 监控 API QPS、响应时间 |
| **自动备份** | 定时 dump MySQL 数据到对象存储 |

## 六、开发节奏建议

### 第一阶段（1~2 个月）：补齐核心体验

优先级排序：
1. 权限与协作模型（成员邀请 + 角色）
2. 全文搜索（MySQL FULLTEXT 快速实现）
3. 页面历史版本
4. 回收站（软删除）

### 第二阶段（1~2 个月）：协作与效率

1. 块级评论
2. 通知系统
3. 模板系统
4. Markdown 导入/导出

### 第三阶段（长期）：高级与生态

1. AI 辅助（接入大模型 API）
2. 第三方集成（OAuth、Webhook）
3. 移动端适配
4. 插件系统
